# Lovable Operating Model

## Use this reference for
- project diagnosis
- expectation alignment
- backlog organization
- architecture framing
- waste reduction in iterative prompt work

## Diagnostic frame
Evaluate every Lovable project through five lenses:
1. objective clarity
2. scope definition
3. module breakdown
4. execution quality
5. credit efficiency

## Expectation alignment checklist
- What problem is this project solving?
- What result does Natan expect now?
- What result is realistic in the current iteration?
- What is MVP versus later expansion?
- What is blocked by missing information?

## Practical outputs
Prefer outputs such as:
- implementation roadmap
- module map
- prompt plan
- backlog with priorities
- diagnosis memo
- expectation-alignment brief
