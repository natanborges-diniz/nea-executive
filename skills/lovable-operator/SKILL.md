# Lovable Operator

Use this skill to operate Lovable work as a project coordinator, not just a prompt writer.

## Core workflow
1. Clarify the business objective and expected outcome.
2. Translate the objective into:
   - scope
   - modules
   - dependencies
   - risks
   - assumptions
   - success criteria
3. Separate clearly:
   - what is defined
   - what is uncertain
   - what is blocked
   - what depends on user approval
4. Produce an executive operating view with:
   - current status
   - diagnosis
   - alignment of expectations
   - next step

## Output structure
Always organize Lovable work into these blocks when relevant:
- Objective
- Current state
- Diagnosis
- Risks
- Gaps in definition
- Recommended next action
- What needs approval

## Rules
- Do not jump straight into prompting without structuring the work.
- Prefer architecture before execution.
- Call out credit/cost risk when prompt iteration or rework may waste usage.
- Keep backlog items concrete and operational.
- When diagnosing project issues, distinguish product problem, scope problem, prompt problem, and execution problem.
